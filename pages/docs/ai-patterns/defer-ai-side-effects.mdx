# Defer AI side effects

This guide walks you through adding deferred side effects to an AI agent function. You will define a deferred function, call it from your agent, register it in the serve handler, and verify it in the Inngest dashboard.

For the general concept of deferred functions, see [Deferred functions](/docs/guides/deferred-functions).

## Prerequisites

- Inngest TypeScript SDK v4 (`npm install inngest@latest`)
- An existing Inngest function that calls an LLM
- Familiarity with `step.run()` and `inngest.createFunction()`

---

## 1. Define a deferred function

Create a deferred function that runs after your agent completes. This example tracks token costs per customer.

```ts
import { createDefer } from "inngest/experimental";
import { z } from "zod";

export const trackCosts = createDefer(inngest, {
  id: "track-ai-costs",
  schema: z.object({
    model: z.string(),
    promptTokens: z.number(),
    completionTokens: z.number(),
    customerId: z.string(),
  }),
}, async ({ event, step }) => {
  await step.run("log-cost", async () => {
    const costPer1k = event.data.model === "gpt-4o" ? 0.005 : 0.00015;
    const totalTokens = event.data.promptTokens + event.data.completionTokens;

    await analytics.track("ai.cost", {
      customer_id: event.data.customerId,
      cost_usd: (totalTokens / 1000) * costPer1k,
      model: event.data.model,
    });
  });
});
```

The `schema` defines the typed payload. When you call `defer()` from the parent, the data is validated against this schema at the call site.

---

## 2. Call defer from your agent function

Add a `defer()` call after the agent work is done. The agent returns its response to the user. Cost tracking runs after the parent finalizes.

```ts
const handleTicket = inngest.createFunction(
  { id: "handle-ticket", triggers: { event: "support/ticket.created" } },
  async ({ event, step, defer }) => {
    const response = await step.run("call-model", async () => {
      return await openai.chat.completions.create({
        model: "gpt-4o",
        messages: [
          { role: "system", content: "You are a support agent." },
          { role: "user", content: event.data.content },
        ],
      });
    });

    const reply = response.choices[0].message.content;

    await step.run("send-reply", async () => {
      await support.reply(event.data.ticketId, reply);
    });

    // Track token costs after the parent completes
    defer("track-costs", {
      function: trackCosts,
      data: {
        model: "gpt-4o",
        promptTokens: response.usage.prompt_tokens,
        completionTokens: response.usage.completion_tokens,
        customerId: event.data.customerId,
      },
    });

    return { ticketId: event.data.ticketId, status: "resolved" };
  }
);
```

`defer()` is synchronous. It returns `void`. The parent continues to the return statement without waiting.

---

## 3. Register in the serve handler

Add both the agent function and the deferred function to your serve handler. Deferred functions are regular Inngest functions and must be registered to receive runs.

```ts
import { serve } from "inngest/next";
import { inngest } from "./client";
import { handleTicket } from "./functions/handle-ticket";
import { trackCosts } from "./functions/track-costs";

export const { GET, POST, PUT } = serve({
  client: inngest,
  functions: [handleTicket, trackCosts],
});
```

## 4. Add more deferred functions

You can call `defer()` multiple times in one parent. Each call creates an independent run. Add a Slack notification alongside cost tracking:

```ts
export const notifyTeam = createDefer(inngest, {
  id: "notify-agent-completion",
  schema: z.object({
    channel: z.string(),
    summary: z.string(),
    ticketId: z.string(),
    model: z.string(),
  }),
}, async ({ event, step }) => {
  await step.run("post-to-slack", async () => {
    await slack.chat.postMessage({
      channel: event.data.channel,
      text: `Agent resolved ticket ${event.data.ticketId} using ${event.data.model}:\n>${event.data.summary}`,
    });
  });
});
```

Add the second `defer()` call to the parent:

```ts
// After the existing defer("track-costs", ...) call
defer("notify", {
  function: notifyTeam,
  data: {
    channel: "#support-resolved",
    summary: reply.slice(0, 200),
    ticketId: event.data.ticketId,
    model: "gpt-4o",
  },
});
```

Register `notifyTeam` in the serve handler alongside the others.

---

## 5. Verify in the dashboard

Open the Inngest dashboard and navigate to the parent run's trace view. You will see the parent run's steps (call-model, send-reply) and the deferred runs linked below it. Each deferred run has its own trace with its own steps and retry history.

The parent run's status is "completed" regardless of whether the deferred runs succeed. A failed notification does not affect the parent.
