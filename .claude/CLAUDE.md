# Docs Writing Voice

Voice validated on the Next.js quickstart rewrite (PR #1495, 73% reduction in time-to-signup). Use this for all docs authoring.

**Sentences:** Short, declarative. One idea per sentence. No compound sentences with semicolons. Paragraphs 1-3 sentences.

**Openings:** Lead with what you're about to do, not why. "This tutorial walks you through adding Inngest to a Next.js app." Not "Inngest is a powerful platform that enables..."

**Framing:** Task-oriented. The reader is doing something, not learning about something.

**Code:** Comes immediately after a one-sentence setup. No multi-paragraph explanation before the code block. Code speaks first.

**Transitions:** Horizontal rules between major sections. No transitional prose like "Now that we've set up X, let's move on to Y."

**Voice markers:** Second person, present tense for instructions, no filler words (simply, just, easily, powerful, seamless), no em dashes, no exclamation points, no hedging, minimal adjectives.

**Structure:** Prerequisites once at the top. Numbered steps for sequential actions. Code groups for framework variants. "Next steps" at the end with bold labels.

**Avoid:** No screenshots unless showing dashboard UI. No expandable sections. No "learn more" asides mid-flow. No philosophical explanations. No comparisons to other tools. If it reads like "Claude definitely wrote this," rewrite it.
